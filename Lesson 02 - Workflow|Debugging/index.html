NodeJS Workflow and Debugging

In this section, I learned about setting up the workflow of our node.js application and debugging errors that we face during the development of our application.

NPM Scripts

We use npm scripts to run certain commands for our application that perform certain specific tasks. For example we can have a script that runs our application, or runs unit tests for our code as well as building our application into a bundled minified production folder(such as in vuejs). To write our scripts we first need to create our package.json file. To create it we run npm init. This generates are file with content like below: 

  1. {
  2.   "name": "lesson-02",
  3.   "version": "1.0.0",
  4.   "description": "Example for lesson 02",
  5.   "main": "app.js",
  6.   "scripts": {
  7.     "serve": "node app.js", // added after package.json file is generated
  8.     "test": "echo \"Error: no test specified\" && exit 1"
  9.   },
  10.   "author": "Peter Kitonga",
  11.   "license": "ISC"
  12. }

In the above example we can see that a script called "serve" is added after the file(package.json) is generated. This allows us to run the node application by running the command npm run serve.

3rd Party Packages

In nodejs we can use thrid party packages that are maintained in the npm repositories. These packages are installed into our application via the command npm install [package name] --save. This installs the third party package and can be seen in the package.json file under the dependencies object. This means the package is now available for production use in our application. We can also install packages by using the --save-dev flag which puts our package under the devDependencies object which simply means they are dev packages for our local development and testing use. The purpose of this is to make clear which package is used for what.

Debugging

For debugging, we used the vscode debugging tools to debug our code by setting breakpoints and manually stepping through our code to this breakpoints. For more info on this please visit the official vscode documentation.

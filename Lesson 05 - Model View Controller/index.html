MVC Architecture

The Model View Controller Architecture

In this section, I learned about the MVC architecture. MVC stands for Models, Views, Controllers. MVC is all about seperation of concerns of our code.

Controllers

Controllers serve as a connection between our Models and Views. They basically contain the in between logic. In nodejs, these controllers can be used to define middleware logic for our routes and therefore also serve as logic for our routes. In the below example, I have moved all the shop logic for our routes to a controller called products.js 

  1. const products = []
  2.  
  3. const getAddProduct = (req, res, next) => {
  4.     res.render('products/add', { 
  5.         page_title: 'Shop | Add Product', 
  6.         route_name: 'products.add'
  7.     })
  8. }
  9.  
  10. const postAddProduct = (req, res, next) => {
  11.     let { title } = req.body
  12.     products.push({ title })
  13.  
  14.     res.redirect('/')
  15. }
  16.  
  17. const getProducts = (req, res, next) => {
  18.     res.render('shop', { 
  19.         prods: products, 
  20.         page_title: 'Shop | Home', 
  21.         route_name: 'home'
  22.     })
  23. }
  24.  
  25. module.exports = { getAddProduct, postAddProduct, getProducts }

Now all we need is to import our controller in the routes file and pass the respective exports for their respective routes. 

  1. // shop.js
  2. const express = require('express')
  3.  
  4. const products_controller = require('../controllers/products')
  5.  
  6. const router = express.Router()
  7.  
  8. router.get('/', products_controller.getProducts)
  9.  
  10. module.exports = { shop_routes: router }
  11.  
  12. // admin.js 
  13. const express = require('express')
  14.  
  15. const router = express.Router()
  16.  
  17. const products_controller = require('../controllers/products')
  18.  
  19. router.get('/products/add', products_controller.getAddProduct)
  20.  
  21. router.post('/products/store', products_controller.postAddProduct)
  22.  
  23. module.exports = { admin_routes: router }
Models

Models serve as a representation of our data in our code. They work by manipulating data to and from a database such as saving, fetching and updating. Models are often defined as classes and the naming convention is to name them in singular with the first character in uppercase. Below is an example model created for our dummy products called Product in the file models/product.js. 

  1. const products = [];
  2.  
  3. class Product {
  4.     constructor(title) {
  5.         this.title = title;
  6.     }
  7.  
  8.     save() {
  9.         products.push(this);
  10.     }
  11.  
  12.     /**
  13.      * We use static here so that we can access the method directly
  14.      * without first instatiating the class
  15.     */
  16.     static fetchAll() {
  17.         return products;
  18.     }
  19. }
  20.  
  21. module.exports = Product;

Now we will import the model in controllers/products.js like so: 

  1. const Product = require('../models/product');
  2.  
  3. const getAddProduct = (req, res, next) => {
  4.     res.render('products/add', { 
  5.         page_title: 'Shop | Add Product', 
  6.         route_name: 'products.add'
  7.     });
  8. };
  9.  
  10. const postAddProduct = (req, res, next) => {
  11.     let { title } = req.body;
  12.  
  13.     const product = new Product(title);
  14.     product.save();
  15.  
  16.     res.redirect('/');
  17. };
  18.  
  19. const getProducts = (req, res, next) => {
  20.     const products = Product.fetchAll();
  21.  
  22.     res.render('shop', { 
  23.         prods: products, 
  24.         page_title: 'Shop | Home', 
  25.         route_name: 'home'
  26.     });
  27. };
  28.  
  29. module.exports = { getAddProduct, postAddProduct, getProducts };
Try it

To run the app, in your terminal run the following commands: 

  1. cd assets/scripts/lesson-05
  2.  
  3. npm install // if you have not installed dependencies
  4.  
  5. npm run serve // starts application

Deno, CRUD & Databases

In this section, I learned about how to connect to and query a database. I used a 3rd party modules called deno_mongo to establish a connection to the mongodb database. I was also able to implement a simple REST API using another 3rd party module called oak that is inspired by node.js's koa framework.

Setting up connection to database

To connect to a mongodb database we need to initialize the MongoClient and pass the connection string to the connect() method. This provides cursors that can be used to connect to specific collections and query them. 

  1. import {
  2.   MongoClient
  3. } from "https://deno.land/x/mongo@v0.23.1/mod.ts";
  4. import {
  5.   Database
  6. } from "https://deno.land/x/mongo@v0.23.1/src/database.ts";
  7.  
  8. const client = new MongoClient();
  9. const databaseName = Deno.env.get("MONGO_DATABASE") ? ? "deno_learning_api";
  10.  
  11. export const connectDatabase = async(): Promise < Database > => {
  12.   try {
  13.     let connection: Database;
  14.     const databaseHost = Deno.env.get("MONGO_HOST");
  15.     const databasePort = Number(Deno.env.get("MONGO_PORT")) ? ? 27017;
  16.     const databaseProvider = Deno.env.get("MONGO_PROVIDER");
  17.     const databaseUsername = Deno.env.get("MONGO_USERNAME");
  18.     const databasePassword = Deno.env.get("MONGO_PASSWORD");
  19.  
  20.     if (databaseProvider === "atlas") {
  21.       /**
  22.        * Connecting to a Mongo Atlas Database
  23.        * 
  24.        * Note: will only work with tiers that support cursors
  25.        */
  26.       connection = await client.connect(
  27.         `mongodb+srv://${databaseUsername}:${databasePassword}@${databaseHost}/${databaseName}?authMechanism=SCRAM-SHA-1&retryWrites=true&w=majority`
  28.       );
  29.     } else {
  30.       // Connecting to a Local Database
  31.       connection = await client.connect(
  32.         `mongodb://${databaseUsername}:${databasePassword}@${databaseHost}:${databasePort}/?authSource=admin`
  33.       );
  34.     }
  35.  
  36.     return connection;
  37.   } catch (err) {
  38.     throw err;
  39.   }
  40. };
  41.  
  42. export const getDatabase = (): Database => {
  43.   try {
  44.     const db = client.database(databaseName);
  45.  
  46.     return db;
  47.   } catch (err) {
  48.     throw err;
  49.   }
  50. };
Qeurying a collection

To query a collection, we first need to import the getDatabase function created in the above step. This gives access to cursors that we can use to query a collection. 

  1. interface TodoSchema {
  2.   _id: {
  3.     $oid: string;
  4.   };
  5.   text: string;
  6. }
  7.  
  8. export const getTodos = async(ctx) => {
  9.   try {
  10.     const allTodos = await getDatabase()
  11.       .collection<TodoSchema>("todos")
  12.       .find()
  13.       .toArray();
  14.     const result = allTodos.map((todoItem) => {
  15.       return {
  16.         id: todoItem._id,
  17.         text: todoItem.text
  18.       };
  19.     });
  20.  
  21.     return (ctx.response.body = {
  22.       todos: result
  23.     });
  24.   } catch (err) {
  25.     console.log(err);
  26.   }
  27. };
Try it

To run the app, in your terminal run the following commands: 

  1. // frontend
  2. cd assets/scripts/lesson-32/frontend
  3.  
  4. npm install // if you have not installed dependencies
  5.  
  6. npm start // starts application
  7.  
  8. // deno API
  9. cd assets/scripts/lesson-32/backend
  10.  
  11. cp .env.example .env // copy env variables into variable file and change their values
  12.  
  13. /**
  14.   * Starts application
  15.   *
  16.   * --allow-net grants permission for the server to be accessed through a port
  17.   * --allow-env grants permission for the env variables
  18.   * --allow-read grants permission for reading files(helpful for reading env variables from .env file)
  19.   * -r forces deno to re-download the modules to the cache(optional)
  20.   */
  21. deno run --allow-net --allow-env --allow-read --allow-plugin --unstable src/app.ts

Working with REST APIs - Practical

In this section, I refresh my knowledge on REST(Representational State Transfer) APIs. REST APIs are useful to provide access to data that is not rendered into HTML pages from a server-side application. They provide data to mobile apps, single page apps and even when accessing a 3rd party API like google maps.

Stateless Authentication

REST APIs by convention are stateless which means there are no sessions stored for authentication of users. To account for this we use Bearer authentication generated from the server-side that are of two types: JWT(JSON Web Token) and Oauth 2. Read here → for information on the differences between the two. In this example we use JWT where we first install the jsonwebtoken(npm install --save jsonwebtoken) dependency. Here we use the .sign() method to generate a token that will be sent back as a response during login. 

  1. // controllers/auth.js
  2. const jwt = require('jsonwebtoken');
  3.  
  4. exports.login = (req, res, next) => {
  5.   const {
  6.     email,
  7.     password
  8.   } = req.body;
  9.   const errors = validationResult(req);
  10.  
  11.   if (!errors.isEmpty()) {
  12.     const error = new Error('Validation failed, please check your inputs');
  13.     error.status_code = 422;
  14.     error.data = errors.array();
  15.     throw error;
  16.   }
  17.  
  18.   User.findOne({
  19.     email
  20.   }).then(user => {
  21.     // nesting the promise because we dont want this logic to run when a user is not found
  22.     return bcrypt.compare(password, user.password).then(matched => {
  23.       // if password is a match, generate a token
  24.       if (matched) {
  25.         // generate token
  26.         const token = jwt.sign({
  27.             email: user.email,
  28.             userId: user._id.toString()
  29.           },
  30.           process.env.JWT_SECRET, // secret string stored in the server-side for signature verification
  31.           {
  32.             expiresIn: '1h'
  33.           } // expiration of token set to one hour
  34.         );
  35.  
  36.         res.status(200).json({
  37.           status: 'success',
  38.           data: {
  39.             access_token: token,
  40.             userId: user._id.toString()
  41.           }
  42.         });
  43.       } else {
  44.         const error = new Error('The password you entered is incorrect.');
  45.         error.status_code = 401;
  46.         throw error;
  47.       }
  48.     }).catch(err => {
  49.       if (!err.status_code) {
  50.         err.status_code = 500;
  51.       }
  52.  
  53.       next(err);
  54.     });
  55.   }).catch(err => {
  56.     if (!err.status_code) {
  57.       err.status_code = 500;
  58.     }
  59.  
  60.     next(err);
  61.   });
  62. };

Once the token is generated, all that is needed is to pass the token to the Authorization header like so: Authorization: Bearer [TOKEN]. The server will use this token to determine which user is accessing the resource. To do this we can have a middleware placed in all routes that require authentication that decodes the token to fetch the user id. For this we use the .verify() method where we pass the token and the secret string stored in our .env file. 

  1. // middlewares/auth-check.js
  2. const jwt = require('jsonwebtoken');
  3.  
  4. module.exports = (req, res, next) => {
  5.   const auth_header = req.get('Authorization');
  6.  
  7.   if (!auth_header) {
  8.     const error = new Error('Unauthenticated request. Please login.');
  9.     error.status_code = 401;
  10.     throw error;
  11.   } else {
  12.     const token = auth_header.split(' ')[1];
  13.     let decoded_token;
  14.  
  15.     try {
  16.       decoded_token = jwt.verify(token, process.env.JWT_SECRET);
  17.  
  18.       if (!decoded_token) {
  19.         const error = new Error('Unauthenticated request. Please login.');
  20.         error.status_code = 401;
  21.         throw error;
  22.       }
  23.  
  24.       req.userId = decoded_token.userId;
  25.  
  26.       next();
  27.     } catch (err) {
  28.       err.status_code = 500;
  29.       throw err;
  30.     }
  31.   }
  32. };

  5.  
  6. npm start // starts application
  7.  
  8. // backend
  9. cd assets/scripts/lesson-22/backend
  10.  
  11. npm install // if you have not installed dependencies

Sessions & Cookies

Sessions & Cookies

In this section, I learned how to create, fetch and remove sessions and cookies. They are used to store selected data such as authentication status/user data e.t.c. It is not advisable to store sensitive data in cookies such as user credentials or payment info because they are visible on the browser and can be easily manipulated on the client-side. This is where sessions have an advantage since they are stored and managed on the server-side.

Creating & Fetching Cookies

Cookies are created by setting the Set-Cookie header. When this header is present in the response, the client browser will store the value of the cookie in it's storage. Once the cookie is stored, it is always passed as a header by default to every request performed. Cookies are mostly used to store authentication tokens where we set an expiration time so as to prevent unwanted use of the token if stolen. 

  1. // controllers/auth.js
  2. const authenticate = (req, res, next) => {
  3.   res.setHeader('Set-Cookie', 'authenticated=true; HttpOnly');
  4.   res.redirect('/');
  5. };
  6.  
  7. // controllers/shop.js
  8. const getHome = (req, res, next) => {
  9.   let authenticated = req.get('Cookie').split(';')[0].trim().split('=')[1];
  10.  
  11.   Product.find().then(products => {
  12.     res.render('shop/index', {
  13.       prods: products,
  14.       page_title: 'Shop | Home',
  15.       route_name: 'home',
  16.       authenticated
  17.     });
  18.   }).catch(err => console.log(err));
  19. };
Creating & Fetching Sessions

Sessions are stored in the server-side and are often used together with cookies. For example we can store a hashed session id in a cookie which the server will use to check and fetch the appropriate session data. Here we the use the express-session plugin. Before using sessions we first need to add a session middleware in the app.js file. This is where we pass configurations for all sessions. The session data can then be store in the request for access by all middlewares. 

  1. // app.js
  2. const express = require('express');
  3. const session = require('express-session');
  4.  
  5. const app = express();
  6.  
  7. app.use(session({
  8.   secret: 'Otooniefi4eYeilah0IMoofohv3koobiequie9ieg3Sa0ia9vie9oefahl2l',
  9.   resave: false,
  10.   saveUninitialized: false
  11. }));
  12.  
  13. // controllers/auth.js
  14. const authenticate = (req, res, next) => {
  15.   // stores session data in the request
  16.   req.session.authenticated = true;
  17.   res.redirect('/');
  18. };
  19.  
  20. // controllers/shop.js
  21. const Product = require('../models/product');
  22.  
  23. const getHome = (req, res, next) => {
  24.   Product.find().then(products => {
  25.     res.render('shop/index', {
  26.       prods: products,
  27.       page_title: 'Shop | Home',
  28.       route_name: 'home',
  29.       authenticated: req.session.authenticated
  30.     });
  31.   }).catch(err => console.log(err));
  32. };
Using MongoDB to store Sessions

By default sessions are configured to be store in memory. This however is not ideal because disk space will eventually get filled up. To deal with this we can store session data in our MongoDB database. We first need to install a dependency called connect-mongodb-session(npm install --save connect-mongodb-session). This will allow us to configure the collection that will be used to store the session data and other useful configurations. 

  1. // app.js
  2. const express = require('express');
  3. const session = require('express-session');
  4. const MongoDBStore = require('connect-mongodb-session')(session);
  5.  
  6. const app = express();
  7. dotenv.config(); // loads the .env variables
  8.  
  9. // storing the mongodb uri for reuse
  10. const mongodb_uri = `mongodb+srv://${process.env.MONGO_USERNAME}:${process.env.MONGO_PASSWORD}@nodejs.j6dzr.mongodb.net/${process.env.MONGO_DATABASE}?retryWrites=true&w=majority`;
  11.  
  12. // creates the store which we can pass to the session middleware below
  13. const store = new MongoDBStore({
  14.   uri: mongodb_uri,
  15.   collection: 'sessions'
  16. });
  17.  
  18. // session middleware
  19. app.use(session({
  20.   secret: 'Otooniefi4eYeilah0IMoofohv3koobiequie9ieg3Sa0ia9vie9oefahl2l',
  21.   resave: false,
  22.   saveUninitialized: false,
  23.   store
  24. }));
Storing user data in Sessions

We can store data in the sessions collection after configuring the store in the above section. This will allow us to know to which user the session belongs to. This session data can then be used to fetch the authenticated user data. 

  1. // app.js
  2. const express = require('express');
  3. const session = require('express-session');
  4.  
  5. const app = express();
  6.  
  7. // middleware that fetches the user data as a mongodb object
  8. app.use((req, res, next) => {
  9.   if (!req.session.user) {
  10.     next();
  11.   } else {
  12.     // get user from session
  13.     User.findById(req.session.user._id).then(user => {
  14.         req.user = user;
  15.  
  16.         next();
  17.       })
  18.       .catch(err => console.log(err));
  19.   }
  20. });
  21.  
  22. // controllers/auth.js
  23. const User = require('../models/user');
  24.  
  25. const authenticate = (req, res, next) => {
  26.   const {
  27.     email,
  28.     password
  29.   } = req.body;
  30.  
  31.   // fetch user by credentials passed
  32.   User.where({
  33.       email
  34.     }).findOne().then(user => {
  35.       // stores session data in the request
  36.       req.session.authenticated = true;
  37.       req.session.user = user;
  38.  
  39.       // callback ensures we only redirect when the session is successfully stored
  40.       req.session.save((err) => {
  41.         if (err) {
  42.           console.log(err);
  43.         } else {
  44.           res.redirect('/');
  45.         }
  46.       });
  47.     })
  48.     .catch(err => console.log(err));
  49. };
Removing Session data

To remove session data we use the .destroy() method. 

  1. // controllers/auth.js
  2. const logout = (req, res, next) => {
  3.   req.session.destroy((err) => {
  4.     if (err) {
  5.       console.log(err);
  6.     } else {
  7.       res.redirect('/');
  8.     }
  9.   });
  10. };

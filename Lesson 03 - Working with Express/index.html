Working with Express

Express.js Framework

In this section, I learned about using the express.js framework for our nodejs application. This is one of the widely using nodejs frameworks and is designed for building single-page, multi-page and hybrid applications.

Express Middlewares

We define middlewares in express by using the use() method which can also be used to attach other things to express as well. This middleware expects three parameters:

  1. req - the request
  2. res - the response
  3. next - this is a function that allows the request to continue to the next middleware in line
We use these middlewares to perform certain checks before processing a request. For example, check if a user has access to the page being requested. 

  1. const express = require('express');
  2.  
  3. const app = express();
  4.  
  5. /**
  6.  * The next param is a function that allows the request
  7.  * to continue to the next middleware in line
  8. */
  9. app.use((req, res, next) => {
  10.     console.log('In the middleware!')
  11.     next()
  12. });
  13.  
  14. app.use((req, res, next) => {
  15.     console.log('In another middleware!')
  16.     res.send('<h1>Hello from Express!!!!</h1>')
  17. });

In the above example when we access the / route, the request first hits the first middleware then proceeds to the next middleware where we serve a response back. We would not be able to reach the second middleware if the next() function was not passed.

Serving html files

We can serve html files with express by using the sendFile() method provided by the res param of our route middleware. Here all we do is pass the path to our html file. To do this we need to construct the path to our file and for that we use the nodejs core library called path. This utility provides a join() method with which we can join our path for all operating systems since these systems have different ways of defining system paths. 

  1. /**
  2.  * We use the core nodejs library path to
  3.  * construct a path to our html file
  4. */
  5. const path = require('path')
  6. const express = require('express')
  7.  
  8. const router = express.Router()
  9.  
  10. router.get('/', (req, res, next) => {
  11.     /**
  12.      * Here, '__dirname' is refering to the current directory of this routes file.
  13.      * We add '..' to tell nodejs to move one step back in the folder stucture and
  14.      * find views/shop.html
  15.     */
  16.     res.sendFile(path.join(__dirname, '..', 'views', 'shop.html'))
  17. })
  18.  
  19. module.exports = router
Filtering Paths

Whenever we want to a prefixes to routes, we can do this by passing the route prefix as the first parameter in the use method. This will apply the prefix do all routes define in the middlewares. For example we can add the prefix /admin to middlewares we have defined for the admin routes like so: 

  1. // pass routes
  2. // here we can also pass route prefixes for the specified middleware
  3. app.use('/admin', admin_routes)
  4. app.use(shop_routes)
Serving files statically

With express.js we can serve files statically by importing the static middleware. This middleware allows us to serve up files such as images and stylesheets which we can place in a public folder for read access. We do this by passing the path to the public directory in our main app file to the static middleware.

app.js 
  1. // here we load files statically from the public folder
  2. // it gives read access to the files in this folder
  3. app.use(express.static(path.join(__dirname, 'public')))
shop.html 
  1. <head>
  2.     <meta charset="UTF-8">
  3.     <meta http-equiv="X-UA-Compatible" content="IE=edge">
  4.     <meta name="viewport" content="width=device-width, initial-scale=1.0">
  5.     <title>Products Shop</title>
  6.     <link rel="stylesheet" href="/css/main.css">
  7. </head>
Try it

To run the app, in your terminal run the following commands: 

  1. cd assets/scripts/lesson-03
  2.  
  3. npm install // if you have not installed dependencies
  4.  
  5. npm run serve // starts application

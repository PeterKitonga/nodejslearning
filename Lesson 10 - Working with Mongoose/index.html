Working with Mongoose

Working with Mongoose

In this section, I learned how to configure and connect to a MongoDB database using Mongoose. Mongoose is an Object-Document Mapping(ODM) library. This is because MongoDB maintains documents in collections. We use mongoose to create Models and Schemas as well as run queries.

Connecting to the database

To connect to the database, we pass the MongoDB url and pass configuration options as the second parameter to the .connect() method. This returns a promise where we can run the app or catch any errors thrown. 

  1. const dotenv = require('dotenv');
  2. const express = require('express');
  3. const mongoose = require('mongoose');
  4.  
  5. const app = express();
  6. const port = 8180;
  7. dotenv.config(); // loads the .env variables
  8.  
  9. mongoose.connect(
  10.  `mongodb+srv://${process.env.MONGO_USERNAME}:${process.env.MONGO_PASSWORD}@nodejs.j6dzr.mongodb.net/${process.env.MONGO_DATABASE}?retryWrites=true&w=majority`, {
  11.     useNewUrlParser: true,
  12.     useUnifiedTopology: true
  13.   }
  14. ).then(() => {
  15.   console.log('MONGO CONNECTED!');
  16.   // mount the nodejs app to a port
  17.   app.listen(port, () => {
  18.     console.log(`Server running at: http://127.0.0.1:${port}`);
  19.     console.log('Hit CTRL-C to stop the server');
  20.   });
  21. }).catch(err => console.log(err));
Defining a Model

In mongoose we define a model by first defining the schema. Although MongoDB is schemaless, mongoose uses this defined schema to know how data will look in the collection. Thereafter we pass the schema to the model method where we will define the name of the model as the first parameter. 

  1. const {
  2.   Schema,
  3.   model
  4. } = require('mongoose');
  5.  
  6. const product_schema = new Schema({
  7.   title: {
  8.     type: String,
  9.     required: true
  10.   },
  11.   price: {
  12.     type: Number,
  13.     required: true
  14.   },
  15.   description: {
  16.     type: String,
  17.     required: true
  18.   },
  19.   image_url: {
  20.     type: String,
  21.     required: true
  22.   },
  23.   created_at: {
  24.     type: Date,
  25.     default: new Date().toLocaleString('af', {
  26.       timeZone: 'Africa/Nairobi'
  27.     })
  28.   },
  29.   updated_at: {
  30.     type: Date,
  31.     default: new Date().toLocaleString('af', {
  32.       timeZone: 'Africa/Nairobi'
  33.     })
  34.   }
  35. });
  36.  
  37. module.exports = model('Product', product_schema);
Inserting data into database

Mongoose provides a .save() method to save data passed to the model constructor to a collection. This will use the fields described in the schema to format how the data is saved. 

  1. const Product = require('../models/product');
  2.  
  3. const postProduct = (req, res, next) => {
  4.   const {
  5.     title,
  6.     price,
  7.     image_url,
  8.     description
  9.   } = req.body;
  10.  
  11.   const product = new Product({
  12.     title,
  13.     price,
  14.     image_url,
  15.     description
  16.   });
  17.  
  18.   product.save().then(result => {
  19.     res.redirect('/admin/products/all');
  20.   }).catch(err => console.log(err));
  21. };
Fetching data from database

To fetch documents from the products collection using mongoose, we conviniently have a .find() method which can be used like below to fetch all products in the collection. 

  1. const Product = require('../models/product');
  2.  
  3. const getProducts = (req, res, next) => {
  4.   Product.find().then(products => {
  5.     res.render('shop/list', {
  6.       prods: products,
  7.       page_title: 'Shop | Listing',
  8.       route_name: 'list'
  9.     });
  10.   }).catch(err => console.log(err));
  11. };
  12.  
  13. const getProduct = (req, res, next) => {
  14.   const {
  15.     _id
  16.   } = req.params;
  17.  
  18.   Product.findById(_id).then(product => {
  19.       res.render('shop/view', {
  20.         product,
  21.         page_title: 'Shop | Listing | View',
  22.         route_name: 'list.view'
  23.       });
  24.     })
  25.     .catch(err => console.log(err));
  26. };

From the example highlighted we can also see that .findById() is used to fetch a single document where we pass the id as a parameter. This works the same way as .findOne().

Updating data

To update data we using mongoose we can use the .save() method available on the object retrieved by .findById(). This allows for channing of promises where we can process the next action or catch errors if present. 

  1. const Product = require('../models/product');
  2. const moment = require('moment');
  3.  
  4. const updateProduct = (req, res, next) => {
  5.   const {
  6.     _id
  7.   } = req.params;
  8.   const {
  9.     title,
  10.     price,
  11.     image_url,
  12.     description
  13.   } = req.body;
  14.   const current_date = moment().utcOffset(3).format("YYYY-MM-DD HH:mm:ss");
  15.  
  16.   Product.findById(_id).then(product => {
  17.     product.title = title;
  18.     product.price = price;
  19.     product.image_url = image_url;
  20.     product.description = description;
  21.     product.updated_at = current_date;
  22.  
  23.     return product.save();
  24.   }).then(result => {
  25.     res.redirect('/admin/products/all');
  26.   }).catch(err => console.log(err));
  27. };
Deleting data

Here we use the .findByIdAndRemove() method to delete one document with the passed id. 

  1. const Product = require('../models/product');
  2.  
  3. const deleteProduct = (req, res, next) => {
  4.   const {
  5.     _id
  6.   } = req.params;
  7.  
  8.   Product.findByIdAndRemove(_id).then(result => {
  9.       res.redirect('/admin/products/all');
  10.     })
  11.     .catch(err => console.log(err));
  12. };
Using relations in Mongoose

Although in MongoDB we do not have relations(we use embeded documents instead), in Mongoose we can define relations in our models. In the example below, we define a relationship(association) between the products and users collections. This is done on the models like so: 

  1. // models/product.js
  2. const {
  3.   Schema,
  4.   model
  5. } = require('mongoose');
  6. const moment = require('moment');
  7.  
  8. const current_date = moment().utcOffset(3).format("YYYY-MM-DD HH:mm:ss");
  9.  
  10. const product_schema = new Schema({
  11.   title: {
  12.     type: String,
  13.     required: true
  14.   },
  15.   price: {
  16.     type: Number,
  17.     required: true
  18.   },
  19.   description: {
  20.     type: String,
  21.     required: true
  22.   },
  23.   image_url: {
  24.     type: String,
  25.     required: true
  26.   },
  27.   // ref value is the collection we are relating to
  28.   user: {
  29.     type: Schema.Types.ObjectId,
  30.     ref: 'User',
  31.     required: true
  32.   },
  33.   created_at: {
  34.     type: Date,
  35.     default: current_date
  36.   },
  37.   updated_at: {
  38.     type: Date,
  39.     default: current_date
  40.   }
  41. });
  42.  
  43. module.exports = model('Product', product_schema);
  44.  
  45. // models/user.js
  46. const {
  47.   Schema,
  48.   model
  49. } = require('mongoose');
  50. const moment = require('moment');
  51.  
  52. const current_date = moment().utcOffset(3).format("YYYY-MM-DD HH:mm:ss");
  53.  
  54. const user_schema = new Schema({
  55.   name: {
  56.     type: String,
  57.     required: true
  58.   },
  59.   email: {
  60.     type: String,
  61.     required: true
  62.   },
  63.   cart: {
  64.     items: [{
  65.       // ref value is the collection we are relating to
  66.       product: {
  67.         type: Schema.Types.ObjectId,
  68.         ref: 'Product',
  69.         required: true
  70.       },
  71.       quantity: {
  72.         type: Number,
  73.         required: true
  74.       }
  75.     }]
  76.   },
  77.   created_at: {
  78.     type: Date,
  79.     default: current_date
  80.   },
  81.   updated_at: {
  82.     type: Date,
  83.     default: current_date
  84.   }
  85. });
  86.  
  87. module.exports = model('User', user_schema);

Now we can use the relation to store data such as to the user field in the products collection. Below is a middleware in the admin controller(controllers/admin.js). 

  1. const Product = require('../models/product');
  2.  
  3. const postProduct = (req, res, next) => {
  4.   const {
  5.     title,
  6.     price,
  7.     image_url,
  8.     description
  9.   } = req.body;
  10.  
  11.   const product = new Product({
  12.     title,
  13.     price,
  14.     image_url,
  15.     description,
  16.     user: req.user._id // fetched user stored in the request
  17.   });
  18.  
  19.   product.save().then(result => {
  20.     res.redirect('/admin/products/all');
  21.   }).catch(err => console.log(err));
  22. };

We can also query using the relations to get data related to a collection. In the example below we fetch all product data of products in the cart belonging to the request user. We do this using the .populate() method where we pass the path of the desired document items. 

  1. const Product = require('../models/product');
  2.  
  3. const getCart = (req, res, next) => {
  4.   req.user
  5.     .populate('cart.items.product')
  6.     .execPopulate()
  7.     .then(user => {
  8.       const products = user.cart.items;
  9.  
  10.       res.render('shop/cart', {
  11.         products,
  12.         page_title: 'Shop | Cart',
  13.         route_name: 'cart'
  14.       });
  15.     })
  16.     .catch(err => console.log(err));
  17. };

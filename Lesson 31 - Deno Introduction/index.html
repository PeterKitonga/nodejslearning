Introduction to Deno

Introduction to Deno

In this section, I learned about Deno.js. Deno.js is a JavaScript/TypeScript runtime built on Chrome's V8 engine, Rust and Tokio.

Setup and Running Code

Deno does not use npm or node to run code. This means that we need to install the executable globally on our computer or server. Instructions for installing deno can be found here →. With this setup we use the run command to execute any deno code. e.g deno run src/app.ts 

  1. // src/app.ts
  2. const message = "Hi there!";
  3.  
  4. console.log(message);
Permissions

For certain functionality to be executed, Deno requires configuration of permissions. This adds a layer of security because we will only run code that we trust. For example, to right to a file with the logic as below we need to add a flag --allow-write in our command. e.g. deno run --allow-write=message.txt src/app.ts 

  1. // src/app.ts
  2. const text = "This is a test - and it should be stored in a file!!";
  3.  
  4. const encoder = new TextEncoder();
  5. const data = encoder.encode(text);
  6.  
  7. Deno.writeFile("message.txt", data).then(() => {
  8.   console.log("Successfully written to file!");
  9. });
Creating a Web Server

To spin up a server, we first need to import module from the Deno Standard Library. We can as well import 3rd party modules such as dotenv to load our environment variables. Deno embraces modern Javascript and we can easily use the import/export syntax for our imports. Deno also uses async iterables such as for await...of. This allows us to iterate through the server promise objects and send back responses to the client. 

  1. import {
  2.   config
  3. } from "https://deno.land/x/dotenv@v2.0.0/mod.ts";
  4. import {
  5.   serve
  6. } from "https://deno.land/std@0.101.0/http/server.ts";
  7.  
  8. config({
  9.   export: true,
  10.   safe: true
  11. });
  12.  
  13. const port = Number(Deno.env.get("APP_PORT"));
  14. const server = serve({
  15.   port
  16. });
  17.  
  18. /**
  19.  * Async iterables
  20.  * 
  21.  * @link https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/for-await...of
  22.  */
  23. for await (const req of server) {
  24.   req.respond({
  25.     body: "Hello World\n"
  26.   });
  27. }
Try it

To run the app, in your terminal run the following commands: 

  1. // Deno application example
  2. cd assets/scripts/lesson-31
  3.  
  4. cp .env.example .env // copy env variables into variable file and change their values
  5.  
  6. /**
  7.  * Starts application
  8.  *
  9.  * --allow-net grants permission for the server to be accessed through a port
  10.  * --allow-env grants permission for the env variables
  11.  * --allow-read grants permission for reading files(helpful for reading env variables from .env file)
  12.  * -r forces deno to re-download the modules to the cache
  13. */
  14. deno run --allow-net --allow-env --allow-read -r src/app.ts

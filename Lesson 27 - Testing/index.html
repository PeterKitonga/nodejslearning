Testing Node.js Apps

In this section, I refreshed my knowledge on testing node.js applications using mocha and chai.

Setting up Tests

In Node.js, tests are conventionaly setup in a folder called test in the root directory. We can setup tests using one of the most used testing framework called Mocha(npm install --save-dev mocha). This package sets up the structure for organizing tests and configuration of how to run them through a CLI & npm scripts. We can also use a popular assertion library called Chai(npm install --save-dev chai) that tests statements passed to it. We begin by first setting up an npm script(npm test) to run the tests in the test/ folder. This is done using a glob pattern → 

  1. {
  2.   "scripts": {
  3.     "serve": "nodemon app.js",
  4.     "test": "mocha 'test/**/*.spec.js'"
  5.   }
  6. }

With this in place, we can now create a file under the test/specs/ folder called example.spec.js. We use the expect assertion style to test addition of two numbers and evaluate it against the expected value. Here, we use the it statement to organize our "expectations" of the test and is written as a normal English statement. 

  1. const {
  2.   expect
  3. } = require("chai");
  4.  
  5. it("should add numbers correctly", () => {
  6.   const num1 = 2;
  7.   const num2 = 3;
  8.   const result = num1 + num2;
  9.  
  10.   expect(result).to.equal(5);
  11.   expect(result).to.be.a("number");
  12. });
  13.  
  14. it("should not return a result of 6", () => {
  15.   const num1 = 2;
  16.   const num2 = 3;
  17.   const result = num1 + num2;
  18.  
  19.   expect(result).not.to.equal(6);
  20.   expect(result).to.be.a("number");
  21. });
Using stubs

We use stubs to fake results of running code we are using from third party libraries. In the example below we test whether we get a userId after a valid token is decoded using the jsonwebtoken library. For this we use the sinon(npm install --save-dev sinon) library for test doubles(mocks, stubs and spies). This has a plugin for chai called sinon-chai(npm install --save-dev sinon-chai). We create a stub for the verify method and return a "fake" object that contains the userId. Once the expectations run, we clean up the stub by calling restore. 

  1. // test/specs/middlewares/auth-check.spec.js
  2. const chai = require("chai");
  3. const sinon = require("sinon");
  4. const jwt = require("jsonwebtoken");
  5. const sinonChai = require("sinon-chai");
  6.  
  7. const {
  8.   expect
  9. } = chai;
  10. const authMiddleware = require("../../../middlewares/auth-check");
  11.  
  12. chai.use(sinonChai); // imports chai plugin
  13.  
  14. it("should yield a userId after decoding", () => {
  15.   const req = {
  16.     get(headerName) {
  17.       return "Bearer xyz";
  18.     },
  19.   };
  20.  
  21.   const stub = sinon.stub(jwt, "verify").returns({
  22.     userId: "abc"
  23.   });
  24.  
  25.   authMiddleware(req, {}, () => {});
  26.  
  27.   expect(req).to.have.a.property("userId");
  28.  
  29.   expect(stub).to.have.been.calledOnce;
  30.   expect(stub).to.have.been.calledOnceWith("xyz", process.env.JWT_SECRET);
  31.  
  32.   stub.restore();
  33. });
Try it

To run the app, in your terminal run the following commands: 

  1. // rest API example
  2. cd assets/scripts/lesson-27
  3.  
  4. npm install // if you have not installed dependencies
  5.  
  6. cp .env.example .env // copy env variables into variable file and change their values
  7.  
  8. npm run serve // starts application

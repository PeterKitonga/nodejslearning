Arrow Functions

Arrow functions are a syntactic sugar in ES6 where we remove the function keyword and place arrows => before the curly braces {}. The biggest advantage of arrow functions is the use of lexical this which allows us to access values in context in the case of nested functions.

  1. const name = 'Peter';
  2. let age = new Date(Date.now()).getFullYear() - new Date(1993, 10, 09).getFullYear();
  3. const has_hobbies = ['cycling', 'gaming', 'swimming', 'hiking'];
  4.  
  5. const summarizeUser = (user_name, user_age, user_has_hobbies) => {
  6.     return `${user_name} is ${user_age} year old whose hobbies are: ${user_has_hobbies}`;
  7. };
Improved Object Literals

In ES6, definition of object literals has improved where we can pass variable names as object properties. We can also remove the function keyword of methods present which also allows for lexical this.

  1. const gender = 'Male';
  2. const person = {
  3.     gender, // By convention variables as properties are defined first in the object
  4.     name: 'Peter',
  5.     age: 29,
  6.     greeting() {
  7.         // The lexical 'this' is used to access this object and it's properties
  8.         return `Hi, I am ${this.name}`;
  9.     }
  10. }
Primitive and Reference Types

In JavaScript, we have two types of data types:

  1. Primitive Types - These data types include: numbers, strings, booleans, undefined, and null. When you set a variable with this type, the variable is that exact value. When you change the value of that variable, the variable becomes a new value.
  2. Reference Types - These data types include: arrays and objects. This simply means that when you create an object and assign it to a variable, the reference to that value is what is set and not the value itself.

  1. /**
  2.  * Arrays and objects are refered to as 'reference' types. In the case below
  3.  * we get to see that we can modify the hobbies variable even though it is
  4.  * a constant. The reason for this is that reference types only store an 
  5.  * address pointing to the place in memory where an array or object is stored.
  6.  * Which means we have not changed the address but rather it's content.
  7.  * More info: https://itnext.io/javascript-interview-prep-primitive-vs-reference-types-62eef165bec8
  8. */
  9. const hobbies = ['cycling', 'gaming', 'swimming', 'hiking'];
  10. hobbies.push('pencil drawing');
  11. console.log(`Hobbies are: ${JSON.stringify(hobbies)}`);
Rest and Spread Operator

In ES6, we also have the rest and spread operator which are helpful for passing an unknown number of arguments(rest) to a function, concatetation of arrays(spread) and conversion of strings into arrays(spread). 

  1. const fruits = ['oranges', 'grapes', 'mangoes', 'apples'];
  2. const vegetables = ['cucumber', 'kales'];
  3. const shopping = [...vegetables, ...fruits]; // spread operator
  4. console.log(`Shopping list is: ${JSON.stringify(shopping)}`);
  5. // rest operator
  6. const checkVegetables = (...vegetables) => {
  7.     return `Arguments passed are: ${vegetables}`;
  8. };
  9. console.log(checkVegetables('carrots', 'tomatoes', 'peppers', 'onions'));
Destructuring

In ES6, destructuring provides an intuitive and flexible way of unpacking values from arrays and properties from objects into individual variables during assignment. 

  1. const furniture = {
  2.     name: 'Waterfall Table',
  3.     type: 'table',
  4.     price: 'KES 2000'
  5. };
  6. const furniture_types = ['table', 'chair', 'bed'];
  7.  
  8. const { price } = furniture; // object destructuring
  9. const [table, chair] = furniture_types; // array destructuring
  10.  
  11. const printName = ({ name }) => {
  12.     return name;
  13. };
  14. console.log(`The furniture name is: ${printName(furniture)}`);
Promises and Async

In ES6, a Promise is a proxy for an unknown value of an asynchronous action. It allows us to associate handlers for an eventual success value or failure reason. We use promises to supply the value of an asynchronous method(mostly REST API requests) at some point in the future. There are tree states of a Promise:

  1. pending - initial state, neither fulfilled nor rejected
  2. fulfilled - meaning the operation was completed successfully. This is handled by the then() block.
  3. rejected - meaning the operation failed. This is usually handled by the catch() block.

  1. const fetchText = () => {
  2.     return new Promise((resolve, reject) => {
  3.         setTimeout(() => {
  4.             resolve('Some text!');
  5.         }, 1500);
  6.     });
  7. }
  8.  
  9. fetchText()
  10.     .then(text => {
  11.         console.log(`First instance of fetchText() result: ${text}`);
  12.  
  13.         return fetchText(); // we can chain the fetch text promise to be ran again
  14.     })
  15.     .then(text => {
  16.         console.log(`Second instance of fetchText() result: ${text}`);
  17.     });

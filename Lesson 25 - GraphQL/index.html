Working with GraphQL

Working with GraphQL

In this section, I learned about GraphQL which a query language for APIs. It offers flexibility in how we query data from APIs removing complexities that REST APIs create as a project grows. When working with GraphQL, we only need one endpoint which provides us with all our data requirements.

Setting up a simple query

To begin querying with GraphQL we first need to install two libraries: graphql(npm install --save graphql) and express-graphql(npm install --save express-graphql). We first need to setup the schema of our query like below: 

  1. // graphql/schema.js
  2. const {
  3.   buildSchema
  4. } = require('graphql');
  5.  
  6. module.exports = buildSchema(`
  7.     type TestData {
  8.         text: String!
  9.         views: Int!
  10.     }
  11.  
  12.     type Query {
  13.         hello: TestData!
  14.     }
  15.  
  16.     schema {
  17.         query: Query
  18.     }
  19. `);

With this in place we then need to define a resolver with a similar name as the query type definition. This resolver is what contains the data to be return as a response. 

  1. // graphql/resolvers.js
  2. module.exports = {
  3.   hello() {
  4.     return {
  5.       text: 'Hello World!',
  6.       views: 1111
  7.     };
  8.   }
  9. }

We can now use the above files to add a middleware that will listen to all request sent to /graphql. This middleware uses the defined schema and resolver to generate a response with only the parameters we pass. 

  1. const express = require('express');
  2. const {
  3.   graphqlHTTP
  4. } = require('express-graphql');
  5. const graphqlSchema = require('./graphql/schema');
  6. const graphqlResolver = require('./graphql/resolvers');
  7.  
  8. const app = express();
  9.  
  10. app.use('/graphql', graphqlHTTP({
  11.   schema: graphqlSchema,
  12.   rootValue: graphqlResolver
  13. }));

Here we send a query field with the resolver and fields separated with a space of what we want to receive in the response. 

  1. // request body Content-Type as application/json
  2. {
  3.   "query": "{ hello { text } }"
  4. }
  5.  
  6. // response
  7. {
  8.   "data": {
  9.     "hello": {
  10.       "text": "Hello World!"
  11.     }
  12.   }
  13. }
Mutations

In graphql, mutations provide functionality to modify our server-side data. These are technically functions that would performs writes just like in any traditional REST API endpoint. We begin by creating a mutation type in the schema where we can define the parameters of the input data and their respective types as well. 

  1. type User {
  2.   _id: ID!
  3.   name: String!
  4.   email: String!
  5.   password: String
  6.   status: String!
  7.   posts: [Post!]!
  8. }
  9.  
  10. input UserInput {
  11.   email: String!
  12.   name: String!
  13.   password: String!
  14. }
  15.  
  16. type Mutation {
  17.   createUser(user_input: UserInput): User!
  18. }

The above describes what grapql should expect as input data for the mutation and the kind of response to return as the User type. With this in place we can create a resolver with the exact same name as the mutation createUser. This resolver creates a user during the signup process and returns the created user as a response. 

  1. const bcrypt = require('bcryptjs');
  2. const User = require('../models/user');
  3. const validator = require('validator');
  4.  
  5. module.exports = {
  6.   async createUser({
  7.     user_input
  8.   }, req) {
  9.     const {
  10.       email,
  11.       name,
  12.       password
  13.     } = user_input;
  14.     const errors = [];
  15.  
  16.     if (!validator.isEmail(email)) {
  17.       errors.push({
  18.         message: 'The email field should be a valid email.',
  19.         field: 'email'
  20.       });
  21.     }
  22.  
  23.     if (validator.isEmpty(password) || !validator.isLength(password, {
  24.         min: 5
  25.       })) {
  26.       errors.push({
  27.         message: 'The password field should be greater than 5 characters long.',
  28.         field: 'password'
  29.       });
  30.     }
  31.  
  32.     if (errors.length > 0) {
  33.       const error = new Error(`Invalid inputs.`);
  34.       error.data = errors;
  35.       error.code = 422;
  36.       throw error;
  37.     }
  38.  
  39.     const existing = await User.findOne({
  40.       email
  41.     });
  42.  
  43.     if (existing) {
  44.       const error = new Error(`User with email '${email}' already exists`);
  45.       throw error;
  46.     }
  47.  
  48.     const hashed_password = await bcrypt.hash(password, 12);
  49.     const user = new User({
  50.       email,
  51.       name,
  52.       password: hashed_password
  53.     });
  54.  
  55.     const created_user = await user.save();
  56.  
  57.     return { ...created_user._doc,
  58.       _id: created_user._id.toString()
  59.     };
  60.   },
  61.   // other resolvers...
  62. }

Variables

GraphQL variables are the recommended standard practice for passing data to a graphql request. We first begin by passing a name to our query and defining the variables to be used. The types of these variables must much the types passed in the graphql schema. These variables are passed as a second property to the graphql request object. For example below is a query for authenticating a user. 

  1. loginHandler = (event, authData) => {
  2.   event.preventDefault();
  3.  
  4.   const graphql_query = {
  5.     query: `
  6.         query Login($email: String!, $password: String!) {
  7.           login(email: $email, password: $password) {
  8.             token
  9.             userId
  10.           }
  11.         }
  12.       `,
  13.     variables: {
  14.       email: authData.email,
  15.       password: authData.password
  16.     }
  17.   };
  18.  
  19.   fetch('http://127.0.0.1:8180/graphql', {
  20.       method: 'POST',
  21.       headers: {
  22.         'Content-Type': 'application/json'
  23.       },
  24.       body: JSON.stringify(graphql_query)
  25.     })
  26.     .then(res => {
  27.       return res.json();
  28.     })
  29.     .then(resData => {
  30.       if (resData.errors && resData.errors[0].status === 401) {
  31.         throw new Error(resData.errors[0].message);
  32.       }
  33.  
  34.       if (resData.errors) {
  35.         throw new Error("User login failed!");
  36.       }
  37.  
  38.       // do whatever...
  39.     })
  40.     .catch(err => {
  41.       console.log(err);
  42.       // do whatever...
  43.     });
  44. };
File uploads

GraphQL does not natively support file uploads however there are work arounds for handling such requests. Here we can use a dependency called graphql-upload(npm install --save graphql-upload). This library allows us to extend functionality to the express-graphql package for file uploads. Here we add a scalar type which would get the streamed file object from a multipart/form-data request. We first need to modify the rootValue in the app.js file and pass the scalar with a name of our choice(Upload). We map it to the GraphQLUpload class which is an instance of the GraphQLScalarType class. 

  1. const express = require('express');
  2. const {
  3.   graphqlHTTP
  4. } = require('express-graphql');
  5. const graphqlSchema = require('./graphql/schema');
  6. const graphqlResolvers = require('./graphql/resolvers');
  7. const {
  8.   graphqlUploadExpress,
  9.   GraphQLUpload
  10. } = require('graphql-upload');
  11.  
  12. const app = express();
  13.  
  14. app.use(
  15.   '/graphql',
  16.   graphqlUploadExpress({
  17.     maxFileSize: 10000000,
  18.     maxFiles: 10
  19.   }),
  20.   graphqlHTTP({
  21.     schema: graphqlSchema,
  22.     rootValue: {
  23.       Upload: GraphQLUpload,
  24.       ...graphqlResolvers
  25.     },
  26.     // other configs...
  27.   })
  28. );

With this in place, we can use this scalar type in any of our mutations. In the below example, we are uploading a picture for a blog post together with its respective title & content. 

  1. scalar Upload

  3. type Post {

  17.     const {

The below example mutation query follows the specification provided by Jayden Seric here →. It uses a multipart/form-data request where we have a map that describes which files are mapped to which variables. 

  1. uploadHandler = async(event, postData) => {
  2.   event.preventDefault();
  3.  
  4.   const graphql_query = {
  5.     query: `
  6.         mutation PostCreate($image: Upload!, $title: String!, $content: String!) {
  7.           createPost(post_input: {title: $title, content: $content, image: $image}) {
  8.             _id
  9.             title
  10.             content
  11.             creator {
  12.               name
  13.             }
  14.             createdAt
  15.           }
  16.         }
  17.       `,
  18.     variables: {
  19.       image: null,
  20.       title: postData.title,
  21.       content: postData.content
  22.     }
  23.   };
  24.  
  25.   const formData = new FormData();
  26.   formData.append('operations', JSON.stringify(graphql_query));
  27.   formData.append(
  28.     "map",
  29.     JSON.stringify({
  30.       "0": ["variables.image"]
  31.     })
  32.   );
  33.  
  34.   if (postData.image.name !== undefined) {
  35.     formData.append("0", postData.image, postData.image.name);
  36.   }
  37.  
  38.   formData.append('title', postData.title);
  39.   formData.append('content', postData.content);
  40.  
  41.   fetch('http://127.0.0.1:8180/graphql', {
  42.       method: 'POST',
  43.       body: formData,
  44.       headers: {
  45.         'Authorization': `Bearer ${this.props.token}`
  46.       }
  47.     })
  48.     .then(res => {
  49.       return res.json();
  50.     })
  51.     .then(resData => {
  52.       if (resData.errors && resData.errors[0].status === 422) {
  53.         throw new Error(
  54.           "Validation failed. Make sure all fields are filled!"
  55.         );
  56.       }
  57.  
  58.       if (resData.errors) {
  59.         throw new Error("Post creation failed!");
  60.       }
  61.  
  62.       // do whatever...
  63.     })
  64.     .catch(err => {
  65.       console.log(err);
  66.       // do whatever...
  67.     });
  68. };

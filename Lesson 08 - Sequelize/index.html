Understanding Sequelize

In this section, I learned Sequelize which is a promise based Node.js ORM(Object Relational Mapping) library for SQL databases. The library is added via the command npm install --save sequelize.

Connecting to the database

Once we have sequelize installed, we can go ahead and configure the connection to the database. 

  1. const Sequelize = require('sequelize');
  2.  
  3. const sequelize = new Sequelize('nodejs_lessons', 'root', '', {
  4.     dialect: 'mysql',
  5.     host: 'localhost',
  6.     timezone: "+03:00" // for timestamps to use local time
  7. });
  8.  
  9. module.exports = sequelize;
Defining a Model

Once connection is established we can define a model by importing sequelize into our model(Product) file(models/product.js). Here we define the attributes of each column and pass extra configurations such as allowing timestamps. 

  1. const Sequelize = require('sequelize');
  2.  
  3. const sequelize = require('../utils/database');
  4.  
  5. const Product = sequelize.define('product', 
  6. {
  7.     id: {
  8.         type: Sequelize.INTEGER.UNSIGNED,
  9.         autoIncrement: true,
  10.         allowNull: false,
  11.         primaryKey: true
  12.     },
  13.     title: {
  14.         type: Sequelize.STRING,
  15.         allowNull: true
  16.     },
  17.     price: {
  18.         type: Sequelize.DOUBLE,
  19.         allowNull: true
  20.     },
  21.     image_url: {
  22.         type: Sequelize.STRING,
  23.         allowNull: true
  24.     },
  25.     description: {
  26.         type: Sequelize.STRING,
  27.         allowNull: true
  28.     }
  29. }, 
  30. {
  31.     timestamps: true,
  32.     createdAt: 'created_at',
  33.     updatedAt: 'updated_at'
  34. });
  35.  
  36. module.exports = Product;
Inserting data into database with Sequelize

We can now use the Product model to store data to our database using the .create() method which accepts an object of values as a parameter. This method returns a promise that we can use to catch errors and perform an action upon a successful response. 

  1. const postProduct = (req, res, next) => {
  2.     const { title, price, image_url, description } = req.body;
  3.  
  4.     Product.create({
  5.         title,
  6.         price, 
  7.         image_url, 
  8.         description
  9.     }).then(result => {
  10.         res.redirect('/admin/products/all');
  11.     }).catch(err => console.log(err));
  12. };
Fetching data from database with Sequelize

We can also fetch all rows from a table using the defined model. Here we use the .findAll() method which also returns a promise containing the values of our table rows. 

  1. const getProducts = (req, res, next) => {
  2.     Product.findAll().then(products => {
  3.         res.render('shop/list', { 
  4.             prods: products, 
  5.             page_title: 'Shop | Listing', 
  6.             route_name: 'list'
  7.         });
  8.     }).catch(err => console.log(err));
  9. };

To find a specific row by the id, we use the .findByPk() method which returns a promise with our value. 

  1. const getProduct = (req, res, next) => {
  2.     const { id } = req.params;
  3.  
  4.     Product.findByPk(id).then(product => {
  5.         res.render('shop/view', {
  6.             product,
  7.             page_title: 'Shop | Listing | View', 
  8.             route_name: 'list.view'
  9.         });
  10.     })
  11.     .catch(err => console.log(err));
  12. };
Updating data using Sequelize

We can use the .save() method to update our data like below. However, we must fetch the desired row by id using the .findByPk() method first. 

  1. const updateProduct = (req, res, next) => {
  2.     const { id } = req.params;
  3.     const updated = req.body;
  4.  
  5.     Product.findByPk(id).then(product => {
  6.         product.title = updated.title;
  7.         product.price = updated.price;
  8.         product.image_url = updated.image_url;
  9.         product.description = updated.description;
  10.  
  11.         /**
  12.          * return the save promise to avoid nesting promises. 
  13.          * This will chain the promise result to the then block below
  14.         */
  15.         return product.save();
  16.     })
  17.     .then(result => {
  18.         res.redirect('/admin/products/all');
  19.     })
  20.     .catch(err => console.log(err));
  21. };

Another way to update the row is by using the .update() method like below. 

  1. const updateProduct = (req, res, next) => {
  2.     const { id } = req.params;
  3.     const updated = req.body;
  4.  
  5.     Product.update(updated, {
  6.         where: {
  7.             id
  8.         }
  9.     })
  10.     .then(result => {
  11.         res.redirect('/admin/products/all');
  12.     })
  13.     .catch(err => console.log(err));
  14. };
Deleting data using Sequelize

Below is a snippet where we are deleting an existing product using a combination of the .findByPk() and .destroy() methods. 

  1. const deleteProduct = (req, res, next) => {
  2.     const { id } = req.params;
  3.  
  4.     Product.findByPk(id).then(product => {
  5.         return product.destroy();
  6.     }).then(() => {
  7.         res.redirect('/admin/products/all');
  8.     })
  9.     .catch(err => console.log(err));
  10. };

Alternatively, we can use the .destroy() method only like so. 

  1. const deleteProduct = (req, res, next) => {
  2.     const { id } = req.params;
  3.  
  4.     Product.findByPk(id).then(product => {
  5.         return product.destroy();
  6.     }).then(() => {
  7.         res.redirect('/admin/products/all');
  8.     })
  9.     .catch(err => console.log(err));
  10. };
Defining relationships(associations) in Sequelize

In sequelize we can define relationships between models in various ways. One of these ways is a one-to-one relationship. In the below snippet we define a one-to-one relationship between the users and carts table. This means a user has only one cart and a cart belongs to one user. 

  1. // constraint options are passed as an object
  2. Cart.belongsTo(User, { constraints: true, onDelete: 'CASCADE', foreignKey: {name: 'user_id'} });
  3. User.hasOne(Cart, { constraints: true, onDelete: 'CASCADE', foreignKey: {name: 'user_id'} });

Another way is to define a one-to-many relationship. This is where a parent table has many children and a child belongs to one parent. Below is a snippet where we define the relationship between the users and products table. 

  1. // constraint options are passed as an object
  2. Product.belongsTo(User, { constraints: true, onDelete: 'CASCADE', foreignKey: {name: 'user_id'} });
  3. User.hasMany(Product, { constraints: true, onDelete: 'CASCADE', foreignKey: {name: 'user_id'} });

The other way is to define a many-to-many relationship. This is where a parent table has many children and a child can belongs to multiple parent tables. Here we usually have a separate(pivot) table that contains the foreign keys of these tables. Below is a snippet where we define the relationship between the products and carts table through the cart_items table. This inserts the product_id and cart_id columns in the cart_items table 

  1. // constraint options are passed as an object
  2. Cart.belongsToMany(Product, { through: CartItem, foreignKey: {name: 'cart_id'} });
  3. Product.belongsToMany(Cart, { through: CartItem, foreignKey: {name: 'product_id'} });

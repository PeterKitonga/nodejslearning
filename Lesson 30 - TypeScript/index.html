Using TypeScript in Node.js

In this section, I refreshed my knowledge on TypeScript and how to add support in Node.js applications. TypeScript is a superset of JavaScript that offers great support for type safety and helps us improve the quality of our code during compilation.

Defining Types

In TypeScript, we define types using the key/value pair declaration where the key is the function argument/object key and the value is declared as JavaScript Types such as Number, String, Boolean e.t.c. written in lowecase. 

  1. const add = (n1: number, n2: number, show_result: boolean, phrase: string) => {
  2.   const result = n1 + n2;
  3.  
  4.   if (show_result) {
  5.     console.log(phrase + result);
  6.   } else {
  7.     return result;
  8.   }
  9. };
  10.  
  11. const number_one = 5;
  12. const number_two = 2.8;
  13. const print_result = true;
  14. const result_phrase = 'Result is: ';
  15.  
  16. add(number_one, number_two, print_result, result_phrase);
Type Inference

When assigning a value to a variable TypeScript will automatically infer the type of that variable and so we do not necessarily need to define the type. However, when declaring a variable whose value is to be assigned later, it is good practice to declare the type so that TypeScript will validate the value passed and let us know if the value is an invalid type. 

  1. // declaration of variable to be assigned to a value later
  2. let number_one: number;
  3. const number_two = 2.8;
  4.  
  5. // ...several lines of code
  6.  
  7. // typescript will validate whether it's assigned to actual number
  8. number_one = 5;
Object Types

In TypeScript, we can define an object type by simply listing it's properties and their types. Because of type inference TypeScript will already have infered the types of a declared variable that is an object which means we don't need to define the types. However, as a parameter for a function it is good practice to declare the object type. 

  1. const person: {
  2.   name: string;
  3.   nickname ? : string; // optional property
  4.   age: number
  5. } = {
  6.   name: 'Peter',
  7.   age: 27
  8. };
  9.  
  10. console.log(person);
  11.  
  12. // param here is an object
  13. const printCoord = (pt: {
  14.   x: number;y: number
  15. }) => {
  16.   console.log("The coordinate's x value is " + pt.x);
  17.   console.log("The coordinate's y value is " + pt.y);
  18. };
  19.  
  20. printCoord({
  21.   x: 3,
  22.   y: 7
  23. });
Array Types

In TypeScript, array types are defined by declaring what the type of each value will be. For example, if the values will be an array of strings we use string[]. 

  1. let favourite_activities: string[];
  2. favourite_activities = ['Sports', 'Cooking'];
  3.  
  4. for (const activity of favourite_activities) {
  5.   console.log(activity.toUpperCase());
  6. }
Tuples Types

Tuple types allow you to express an array with a fixed number of elements whose types are known but do not need to be the same. Here we pass the types within an array of the fixed number of types. For example, below we define an array of two elements that has a number and string. 

  1. let roles: [number, string];
  2. roles = [2, 'author'];
Enum Types

Enums allow us to define a set of named constants. Using enums can make it easier to document intent, or create a set of distinct cases. Read more about them here. 

  1. enum Role { ADMIN = 2, READ_ONLY = 100, AUTHOR = 10 };
  2.  
  3. let roles = Role.ADMIN;
  4. console.log(roles);
Union Types

Union types allow a us to define variables whose value may be of different types. We define this by the use of the | symbol where we can define multiple types. 

  1. const combine = (input_one: number | string, input_two: number | string) => {
  2.   let result;
  3.  
  4.   if (typeof input_one === 'number' && typeof input_two === 'number') {
  5.     result = input_one + input_two;
  6.   } else {
  7.     result = `${input_one} ${input_two}`;
  8.   }
  9.  
  10.   return result;
  11. };
  12.  
  13. const combined_ages = combine(30, 26);
  14. console.log(combined_ages);
  15.  
  16. const combine_names = combine('Peter', 'Kitonga');
  17. console.log(combine_names);
Literal Types

Literal types refer to specific strings or numbers as types of a variable. They are defined just like union types using the | symbol. 

  1. // result_conversion is always going to be one of the two defined values
  2. const combine = (input_one: number | string, input_two: number | string, result_conversion: 'as-number' | 'as-text') => {
  3.   let result;
  4.  
  5.   if (typeof input_one === 'number' && typeof input_two === 'number' || result_conversion === 'as-number') {
  6.     // we use the '+' symbol in front of the variables to convert them to a number
  7.     result = +input_one + +input_two;
  8.   } else {
  9.     result = `${input_one} ${input_two}`;
  10.   }
  11.  
  12.   return result;
  13. };
  14.  
  15. const combined_ages = combine(30, 26, 'as-number');
  16. console.log(combined_ages);
  17.  
  18. const combined_string_ages = combine('30', '26', 'as-number');
  19. console.log(combined_string_ages);
  20.  
  21. const combine_names = combine('Peter', 'Kitonga', 'as-text');
  22. console.log(combine_names);
Type Aliases

Type aliases are useful whenever we want to define a type that will used in multiple places in our code. They are also handy for defining union and literal types such as in the example below. 

  1. type Combinable = number | string;
  2. type ConversionDescriptor = 'as-number' | 'as-text';
  3.  
  4. // result_conversion is always going to be one of the two defined values
  5. const combine = (input_one: Combinable, input_two: Combinable, result_conversion: ConversionDescriptor) => {
  6.   let result;
  7.  
  8.   if (typeof input_one === 'number' && typeof input_two === 'number' || result_conversion === 'as-number') {
  9.     // we use the '+' symbol in front of the variables to convert them to a number
  10.     result = +input_one + +input_two;
  11.   } else {
  12.     result = `${input_one} ${input_two}`;
  13.   }
  14.  
  15.   return result;
  16. };
Return Type Annotations

Return types are defined after the parameter list of a function. They define the type of the returned value of a function. In most cases we don't need to define this because TypeScript will infer the type of the value based on the return statements. We can also define a type called void which refers to a function without a return statement. 

  1. const add = (input_one: number, input_two: number): number => {
  2.   return input_one + input_two;
  3. };
  4.  
  5. // void is used when a function doesn't have a return statement
  6. const printResult = (value: number): void => {
  7.   console.log(`Result: ${value}`);
  8. };
  9.  
  10. printResult(add(20, 7));
Function Types

Function types are defined using the keyword Function. However, this will mean that our variable can be assigned to any function as a value. To lock this down, we can define the function type using an anonymous function with the expected parameters(don't have to have the same names) & their types as well as the return type of the function. 

  1. // void is used when a function doesn't have a return statement
  2. const printResult = (value: number): void => {
  3.   console.log(`Result: ${value}`);
  4. };
  5.  
  6. const add = (input_one: number, input_two: number): number => {
  7.   return input_one + input_two;
  8. };
  9.  
  10. printResult(add(20, 7));
  11.  
  12. // function type
  13. let combineValues: (a: number, b: number) => number;
  14. combineValues = add;
  15.  
  16. printResult(combineValues(8, 8));

Callbacks can also be defined with the same annotations as functions where we will use an anonymous function to declare the parameter types and also the return type. 

  1. const addAndHandle = (input_one: number, input_two: number, _callback: (num: number) => void) => {
  2.   const result = input_one + input_two;
  3.  
  4.   _callback(result);
  5. };
  6.  
  7. addAndHandle(10, 8, (result) => {
  8.   // do whatever
  9.   const double = result * 2;
  10.   console.log(`Callback result: ${double}`);
  11. });
The 'unknown' Type

The unknown type is used whenever we don't know the type of the value to be passed to a variable. This means we can pass whatever value we want. This is better than the any type because for unknown we still require to perform a type check before assignment to other variables. 

  1. let user_input: unknown;
  2. let user_name: string;
  3.  
  4. user_input = 5;
  5. user_input = 'Peter';
  6.  
  7. /**
  8.  * Difference of unknown from any type is that we require to do a type check before
  9.  * assigning a variable with an unknown type to a variable with a strict type
  10.  */
  11. if (typeof user_input === 'string') {
  12.   user_name = user_input;
  13. }
The 'never' Type

The never type is used whenever our function never returns a value. This is mostly used when our function throws a custom exception like below. 

  1. const generateError = (message: string, code: number): never => {
  2.   throw {
  3.     message,
  4.     error_code: code
  5.   };
  6. };
  7.  
  8. generateError('An error occured!', 500);
Express Setup

To ensure we work with TypeScript in a Node.js application, we need to install type definitions for express and node(npm install --save-dev @types/express @types/node). These ensure we add static types for express which will allow us to create middlewares with TypeScript. With this in place we can now modify the app.ts file to look like below. 

  1. require('dotenv-expand')(require('dotenv').config());
  2.  
  3. import path from 'path';
  4. import express, {
  5.   NextFunction,
  6.   Request,
  7.   Response
  8. } from 'express';
  9.  
  10. const app = express();
  11. const port = process.env.APP_PORT;
  12.  
  13. app.use(express.json());
  14. app.use(express.static(path.join(__dirname, '../public')));
  15.  
  16. app.use((req: Request, res: Response, next: NextFunction) => {
  17.   res.setHeader('Access-Control-Allow-Origin', '*');
  18.   res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  19.   res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, PATCH, DELETE');
  20.  
  21.   next();
  22. });
  23.  
  24. app.use((err: Error, req: Request, res: Response, next: NextFunction) => {
  25.   res.status(500).json({
  26.     message: err.message
  27.   });
  28. });
  29.  
  30. app.get('/', (req: Request, res: Response, next: NextFunction) => {
  31.   res.status(200).json({
  32.     message: 'Hello!'
  33.   });
  34. });
  35.  
  36. // mount the nodejs app to a port
  37. app.listen(port, () => {
  38.   console.log(`Server running at: ${process.env.APP_BASE_URL}`);
  39.   console.log('Hit CTRL-C to stop the server');
  40. });
Try it

To run the app, in your terminal run the following commands: 

  1. // rest API example
  2. cd assets/scripts/lesson-30
  3.  
  4. npm install // if you have not installed dependencies
  5.  
  6. cp .env.example .env // copy env variables into variable file and change their values
  7.  
  8. npm run compile // compiles typescript code to commonjs
  9.  
  10. npm run serve // starts application

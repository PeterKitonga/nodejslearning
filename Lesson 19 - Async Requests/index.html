Async Requests

In this section, I learn how to implement asynchronous requests for actions where we it is desirable for the client to not reload and perform a request to the server behind the scenes. We do this through the use of json responses.

Delete product functionality with JSON response

We can improve the experience of deleting a product using asynchronous requests where we would receive back a JSON response and remove the element from the DOM with client side JavaScript. Here we can create a file at /public/js/functions.js which will house our client logic like below. We also need to load the file in our admin index view. 

  1. <main>
  2.     <div class="grid">
  3.         <% if (prods.length > 0) { %>
  4.             <% for (let product of prods) { %>
  5.                 <article class="card product-item">
  6.                     <header class="card__header">
  7.                         <h1 class="product__title"><%= product.title %></h1>
  8.                     </header>
  9.                     <div class="card__image">
  10.                         <img src="<%= product.image_url %>" alt="A Book">
  11.                     </div>
  12.                     <div class="card__content">
  13.                         <h2 class="product__price"><%= `$ ${product.price}` %></h2>
  14.                         <p class="product__description"><%= product.description %></p>
  15.                     </div>
  16.                     <div class="card__actions">
  17.                         <a href="/admin/products/edit/<%= product._id %>" class="btn">Edit</a>
  18.  
  19.                         <button type="button" class="btn" onclick="custom.initDeleteProduct(this, '<%= product._id %>', '<%= csrf_token %>')">Delete</button>
  20.                     </div>
  21.                 </article>
  22.             <% } %>
  23.         <% } else { %>
  24.             <h1>No Products</h1>
  25.         <% } %>
  26.     </div>
  27. </main>
  28.  
  29. <script src="/js/functions.js"></script>

In the EJS template above we have a function being called in the onclick function of our delete button. This calls a custom method in the /public/js/functions.js file. Here we use the fetch() method that is natively available in vanilla JavaScript which is used to perform HTTP requests. This method return a promise with a readable stream that we first need to return and fetch in the next then block. 

  1. let custom = {
  2.   initDeleteProduct(btn, product_id, csrf_token) {
  3.     const product = btn.closest('article');
  4.  
  5.     fetch(`/admin/products/delete/${product_id}`, {
  6.         method: 'DELETE',
  7.         headers: {
  8.           'CSRF-Token': csrf_token
  9.         }
  10.       })
  11.       .then(response => {
  12.         return response;
  13.       })
  14.       .then(data => {
  15.         product.parentNode.removeChild(product);
  16.       })
  17.       .catch(err => console.log(result));
  18.   }
  19. };

With that in place, we now modify the deleteProduct route middleware handler in the admin controller. This function will now return a json response after successful deletion. 

  1. // controllers/admin.js
  2. const Product = require('../models/product');
  3. const {
  4.   deleteFile
  5. } = require('../utils/file');
  6.  
  7. const deleteProduct = (req, res, next) => {
  8.   const {
  9.     _id
  10.   } = req.params;
  11.  
  12.   Product.findById(_id).then(product => {
  13.     // ["http://127.0.0.1:8180/", "storage/files/1621249822068-fakeimg.png"] 
  14.     const old_file = product.image_url.split(`${process.env.APP_BASE_URL}/`)[1];
  15.  
  16.     deleteFile(`./public/${old_file}`);
  17.  
  18.     return Product.deleteOne({
  19.       _id,
  20.       user: req.user._id
  21.     });
  22.   }).then(result => {
  23.     res.status(200).json({
  24.       status: 'success'
  25.     });
  26.   }).catch(err => res.status(500).json({
  27.     status: 'error'
  28.   }));
  29. };
Try it

  1. cd assets/scripts/lesson-19
  2.  
  3. npm install // if you have not installed dependencies
  4.  
  5. npm run serve // starts application
